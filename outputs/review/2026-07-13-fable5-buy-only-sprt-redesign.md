# シャドーポートフォリオ再設計: 主検定のBUY限定化（D-021案）

## 0. 結論（要旨）

| 論点 | 結論 |

## 1. 事実確認（読み込んだ一次資料に基づく）

1. **判定区分の規約上の非対称性**（`playbooks/judgement.md`）: BUY判定のみが「エントリー
   価格帯・確信度・投入比率・プロベーション適用比率・シナリオ無効化条件」を明記する規約に
   なっている。WATCHは再評価トリガーと上振れ余地スコア、PASSは見送り判断であり、
   **「この価格で資金を入れる」という提案はBUYだけが行う**。実弾投入の可否・サイズを左右する
   決定（プロベーション卒業=D-011サイジング復帰／KILL=新規実弾停止）の根拠となる検定が
   測るべきは、この「資金を入れる」提案の質である。
2. **現行D-019は全区分を主検定に入力**（`playbooks/shadow-portfolio.md` 統計検定節、
   `corp/tools/shadow_sprt.py` docstring「wins = 全区分の合計」）。設計時の理由は
   (a) BUYが極端に少なく限定すると検証が進まない懸念（設計メモ§7が「BUY固有のエッジは
   未検証のまま」と正直に記録済み）、(b) 母集団の質の測定という別目的の混在、であった。
3. **台帳の現状**（`corp/shadow-portfolio.md`、2026-07-13時点）: SP-001〜SP-014の14件。
   内訳は**BUY 0 / WATCH 8 / PASS 6**。決済済み2件はSP-007（WATCH由来HIT_LOSS）・
   SP-009（PASS由来HIT_PROFIT）で、**いずれもBUY以外からの決済**。現行の主検定入力
   （--wins 1 --losses 1）は全てBUY以外の標本で構成されている。
4. **CEOの指摘の核心**: 「BUYだけが実際の売買を左右するのに、判定を無視した仮想売買
   （WATCH/PASSの即時エントリー）の勝率で判定の精度を検証するのは根本的におかしい」。
   前回レビュー（2026-07-13 timeframe-weighting-review）が提案したIMMEDIATE/DEFER区分は
   「価格トリガーの有無」という別軸での区分であり、BUY/WATCH/PASSという本来の区分とズレる。

事実確認の結論: CEOの指摘は資料と整合する。**実弾規律の切替（卒業/撤退）を左右する主検定に、
実弾投入判断ではないWATCH/PASSの仮想損益を入力する現行設計は、測定対象と意思決定対象が
一致していない**。前回レビューの案Bはこの不一致を「価格トリガー有無」という代理軸で部分的に
緩和するだけで、解消しない。

## 2. IMMEDIATE/DEFER案の独立再検証（結論: 不採用）

前回レビュー（私自身の提案）を既定結論とせず再検証した。

- **抜け穴の確認**: IMMEDIATEの定義は「BUY、および価格水準を容認したWATCH/PASS」だった。
  つまり「今の価格で買っても悪くない」と読めるが実際には買わないWATCH（例: イベント待ち型）や
  PASS（積極的理由なしの見送り）がIMMEDIATE側=主検定に残る。これらは判定関数が
  「資金を入れない」と決めた事例であり、その仮想損益で実弾規律を切り替える転倒は
  解消されない。CEOの指摘通り。
- **代理軸の脆さ**: IMMEDIATE/DEFERの判定は「D-018語彙の価格トリガーの有無＋エントリー
  価格帯の未確定」という文面特徴に依存する。判断メモの書き方の揺れが検定標本の帰属を変える
  （判定区分BUY/WATCH/PASSは統制語彙で確定しており揺れない）。検定標本の定義は
  最も揺れない区分に置くべきである。
- **BUY限定はIMMEDIATE/DEFERの上位互換**: 前回レビューが案Bで守ろうとした実害経路
  （待て型WATCHの負けがKILL_TRIGGERを誤発動→実弾停止）は、BUY限定で完全に遮断される
  （待て型WATCHはそもそも主検定に入らない）。案Bで残る抜け穴もない。
- **部分的に残す価値のある要素**（軸の議論と独立に有効な記録規約。§6で採用）:
  - 共通1: 仮想エントリー・決済判定を「大引け後の確定終値」に統一（D-018の終値主義と整合。
    場中スナップショット依存の再現不能性の解消）。
  - 共通2: 決済損益の二重解釈（検定用=バリア値固定 / 台帳記録=実測終値ベースの参考値。
    バリア超過幅はD-015のギャップ仮定値を実データ化する入力になる。SP-007の-14.08%は
    その初の実測1点）。
- **案C（条件成立日エントリーの並走追跡）も今回は採用しない**: 元々の動機（待て型WATCHの
  正しい成績表を主検定に供給する）はBUY限定化で消滅した。残る価値はD-018トリガー語彙の
  的中率検証だが、これは主検定と無関係の独立テーマであり、記録コストとの見合いで
  CEOが別途要否を判断すべき検討課題として残す（既定採用しない）。

## 3. 新設計: 主検定（BUY限定）と診断指標（全区分）の分離

### 3-1. 主検定（ゲート）

- **入力**: 判定区分=BUYの仮想ポジションのバリア決済結果（HIT_PROFIT/HIT_LOSS）のみ。
  EXPIREDは従来通り標本外（比率25%超で枠組み再検討、BUY系列内で判定）。
- **検定の枠組み**: D-019の二重SPRT（検定G/K・境界lnA=+2.7726/lnB=−1.5581・α=0.05・
  β=0.20・p0=39.2%・pG=50%・pK=28.57%）は**変更しない**。SPRTの境界・作動特性は
  1件ごとの尤度比に基づくため標本の到着頻度に依存せず、入力をBUYに絞っても
  誤り率の保証（実効α=4.9%等）はそのまま成立する。変わるのはカレンダー時間だけである
  （それが甚大であることは§4で正直に示す）。
- **卒業条件の整理**: D-019の卒業3条件のうち(ii)BUYサブセット整合ゲート（BUY≥15件かつ
  BUY勝率≥39.2%）は、主検定自体がBUY限定になったため**吸収・廃止**する（主検定の
  GRADUATE_RECOMMENDはBUY標本のみで出るので、別途のBUY整合チェックは同語反復になる）。
  卒業は (i)検定GのH1採択（BUY標本）＋(ii)CEOの新決定ID発行、の2条件となる。
- **終端の再定義**: D-019の「n=150または12ヶ月の早い方で終端」は週次5〜10件の決済流入を
  前提としていた。BUY限定ではこの前提が崩れるため、**12ヶ月時点を「終端」ではなく
  「定期裁定チェックポイント」に再定義**する: プロベーション開始から12ヶ月ごとに、
  蓄積状況（BUY件数・決済数・LLR・診断指標の参考値）をCEOに報告し、
  継続／条件変更（BUY基準・発掘条件の見直し着手を含む）／撤退をCEOが裁定する。
  n=150到達時の終端判定規則（LLR閾値）は不変。
- **系列の同一性**: 従来通り（BUY基準・使用モデル系列・バリア値の重大変更で系列分割）。
  なお本再設計自体は「どの標本を検定に数えるか」の変更であり、判定関数・バリア・
  登録対象は不変のため、既存BUY標本（現在0件）の系列を分断しない。

### 3-2. 診断指標（全区分ベース・参考値）

- **維持する理由**: WATCH/PASSを含む全登録の無差別仮想エントリーは「発掘条件（D-008）＋
  調査・判断パイプラインが拾ってくる母集団を、判定によらず全部買っていたらどうなるか」を
  測る。これはD-012エッジ仮説の前段（母集団に超過収益の候補が含まれているか）の低コストな
  監視であり、BUYが希少な現状では**唯一データが貯まる系列**でもある。廃止すると
  「BUY 0件の期間は何も観測できない」状態になるため維持する。
- **混同防止のラベリング（義務化）**:
  - 名称を「**参考: 全区分ベース（母集団診断・主検定外）**」に統一する。
  - 週次サマリ・日次ダイジェスト・ダッシュボードでは、主検定（BUY限定）ブロックと
    物理的に別のブロック・別の見出しで記載する。
  - 診断指標には**SPRTの判定語（CONTINUE/GRADUATE_RECOMMEND/KILL_TRIGGER/GRAY_STOP）を
    一切付さない**。記載してよいのは記述統計（件数・勝敗・勝率点推定・EXPIRED・
    区分別内訳）のみ。`corp/tools/shadow_sprt.py`の出力もこの規約に従う（参考ブロックに
    判定を出力しない）。
  - 診断指標がどれほど良くても悪くても、実弾規律（卒業/撤退/サイジング）の切替には
    使わない。使いたくなった場合は新しい決定IDを要する。

## 4. BUY限定の実務帰結（正直な報告）

数値は全て検証スクリプトで再計算可能。前提（週次判断件数・決済ラグ・決済率）は
D-019設計時と同じ**仮定値**であり、実測が乖離すれば更新を要する。

### 4-1. BUY発生率の正直な見積もり（観測: 判断14件中BUY 0件）

| 推定方法 | 値 |
|---|---|
| Clopper-Pearson 片側95%上限 | p_buy ≤ **19.3%** |
| Clopper-Pearson 片側90%上限 | p_buy ≤ 15.2% |
| 「3の法則」近似（3/14） | p_buy ≤ 21.4% |
| Jeffreys事後 Beta(0.5, 14.5) 中央値 | **1.58%** |
| 同 90%点 / 95%点 | 9.1% / 12.6% |

- 観測0/14からは「BUY率の上限」しか言えず、点推定は事前分布に強く依存する。
  Jeffreys中央値1.6%は「観測が続けばこの程度」という一つの目安にすぎない。
- **区間はむしろ楽観側に偏りうる**: 14件は2バッチ（07-09の8件・07-11の6件）で同一地合い・
  同一発掘条件の相関を持ち、iid前提の区間は実際より狭い。また8918・4564・4597等の
  低位株はD-009/D-013のフロア外であり、母集団の構成自体がまだ安定していない。
  逆方向の留意点: 07-09バッチはD-008修正前の崩壊した母集団を含み、D-012の積極的BUY基準は
  運用2週未満である。**将来のBUY率が0/14と同分布である保証はどちらの向きにもない**。

### 4-2. SPRT停止までのカレンダー時間の試算（E[N]≈80件、代表ケース 週8件判断・決済率75%）

| BUY率シナリオ | BUY発生/週 | E[N]=80到達（ラグ込み） | 12ヶ月時点の期待BUY決済数 |
|---|---|---|---|
| 1.6%（Jeffreys中央値） | 0.13件 | **約16年** | 約4件 |
| 2% | 0.16件 | 約13年 | 約5件 |
| 5% | 0.40件 | **約5.2〜5.4年** | 約13件 |
| 10% | 0.80件 | 約2.7〜2.8年 | 約26件 |
| 19.3%（CP95上限） | 1.54件 | **約1.4〜1.6年** | 約50件 |

感度分析（週5〜10件・決済率60〜90%・E[N]=59〜92件）でも、BUY率5%なら3.4〜10年超、
10%でも1.7〜5.1年のレンジであり、**12ヶ月以内の統計的結論はBUY率が20%近い場合ですら
ほぼ不可能**（最速卒業=12連勝の蓄積だけでBUY率5%なら約0.8年＋ラグ、そして12連勝という
系列自体が稀）。参考: 現行D-019（全区分入力）は同条件で0.4〜0.5年であり、設計メモ§2-4の
「4〜7ヶ月」はこの全区分前提の数字だった。

### 4-3. 正直な結論と選択肢の整理

**結論: BUY限定の主検定は、観測と整合する大半のBUY率シナリオで「何年も結論が出ない」。
特に現状のBUY発生ペース（0件/2週）が続くなら、検定は事実上開店休業となる。**
これは設計の失敗ではなく、「実弾判断を左右する検定は実弾判断の標本でしか行えない」という
CEO決定の論理的帰結であり、隠すべきではない事実である。以下の選択肢を整理する
（本レビューの採用案は「a＋c」。bは検討課題としての切り出しのみ）:

- **(a) プロベーション期の長期化を明示的に受け入れる（採用案）**: 実弾は2%サイジング
  （D-019）のまま、卒業も撤退も急がない。リスクは有界（エッジ皆無でも実弾20〜40回の
  期待授業料−1.7〜−3.4%。そもそもBUYが希少なら実弾回数自体が少なく、露出はさらに小さい）。
  12ヶ月ごとのチェックポイント（§3-1）でCEOが継続可否を裁定する。
  帰結として正直に明記すべき点: (1) D-011の通常サイジング（10/5/3%）は数年単位で
  発動しない可能性がある。(2) D-019§3-3の「実弾執行品質の実測で分岐勝率パラメータを
  実データ化する」計画も同じ速度でしか進まない。
- **(b) BUY基準・発掘条件の見直し要否を独立の検討課題として切り出す（CEO裁定事項）**:
  BUY 0/14が「基準が適切で母集団に買うべき銘柄がなかった」のか「D-012基準・D-008発掘条件の
  設計がBUYを構造的に出せない」のかは、現データでは判別できない。**ここでBUY基準の緩和を
  提案することはしない**（検定の母数を稼ぐために投資判断の質を下げるのは本末転倒）。
  切り出す論点は「BUY 0件が判断の正しさの表れか、パイプラインの機能不全か」の診断方法
  （例: 見送り理由の類型集計、発掘条件の各軸からのBUY候補到達率）であり、緩和ありきではない。
- **(c) 診断指標（全区分・参考値）を併記しつつ主検定はBUY限定を堅持する（採用案）**:
  §3-2の通り。BUYが貯まらない期間も母集団の質の監視は継続でき、「何も見えない」状態を避ける。
- （参考・不採用）全区分検定を主検定に戻す: CEOが指摘した測定と意思決定の不一致が復活する
  ため不採用。診断指標として残すことで情報は失わない。

## 5. 既存データの遡及処理（285A・5240）

- SP-007（285A、WATCH由来、HIT_LOSS）・SP-009（5240、PASS由来、HIT_PROFIT）は
  BUY判定ではないため、新設計の主検定の入力から**遡及的に除外**する。
  **主検定はn=0（w=0, l=0）に戻る**。
- 整合性の根拠: 主検定の標本定義は「BUY判定の決済結果」であり、定義変更を過去に一貫して
  適用するのが第2条的に正しい（都合よく片方だけ残す余地がない。なお現在の2件は勝1・負1で
  除外してもLLRへの寄与は小さく、除外の有利不利はほぼ中立である点も付記する）。
- 両件は台帳から削除しない（履歴保存ルール）。診断指標（全区分ベース）の集計には
  勝1・負1として引き続き含まれる。
- 週次サマリ・ダッシュボードで過去に報告した「SPRT入力 w=1, l=1」は、次回の週次サマリで
  「D-021によりBUY限定へ再定義、主検定n=0」と訂正の経緯を明記する。
