# WATCH銘柄台帳（corp/watchlist.md）

初版作成: 2026-07-09（Fable5独立レビュー §1-2 のP1指摘を受けCEOが新設を承認）
位置づけ: 日次ルーチンのイベント検知（D-006、`corp/RUNBOOK.md`）が参照する、WATCH判定銘柄の
一元台帳。再評価トリガーの正本は各判断メモ（「参照元判断メモ」列）だが、日次巡回は
**まず本台帳を参照する**。本台帳と判断メモに食い違いがある場合は判断メモを正とし、本台帳を修正する。

## 運用ルール

- **追加**: 投資判断部の判断メモがWATCHで確定し、リスク管理部の機械照合（APPROVE）を経た時点で、
  銘柄一覧に1行＋再評価トリガー一覧に該当行を追加する（週次ルーチン内で実施。`corp/RUNBOOK.md` 参照）。
- **日次更新**: 日次ルーチン2（イベント検知）の実行者が、巡回した銘柄の「最終確認日」を更新する。
  トリガー成立を検知した場合はトリガーの「状態」を `OPEN` → `TRIGGERED` に変更し、当日中に
  イベント駆動ルーチンへ引き継ぐ（トリガー行は削除しない）。
- **解除**: WATCH解除（BUY/PASSへの判定変更）は、投資判断部の**新しい判断メモ**（1稿＋1校閲）に
  よってのみ行う。解除時は銘柄の「状態」を `ACTIVE` → `CLOSED` に変更し、解除根拠となった
  判断メモのパスを「参照元判断メモ」列に追記する。**行の削除はしない**（履歴として残す）。
- **構造的除外による解除（例外）**: 投資判断の見直しではなく、制度上そもそも売買が成立しない
  ことが判明した場合（例: D-013の単元株フィージビリティにより1単元すら購入不能と裁定された
  銘柄）は、新しい判断メモを経ずに`CLOSED`化してよい。この場合「参照元判断メモ」列には
  元の判断メモに加えて、根拠となったD-log決定（例: D-013）を併記する。
- 状態の統制語彙 — 銘柄: `ACTIVE`（監視中）/ `CLOSED`（解除済み）。
  トリガー: `OPEN`（未成立）/ `TRIGGERED`(成立・イベント駆動ルーチンへ引き継ぎ済み) /
  `EXPIRED`（予定日経過等で失効）。
- 本台帳の数値・条件は各判断メモからの転記であり、新規取得数値を含まない（第2条の出典・取得日時は
  参照元判断メモに記録済み）。

## 容量上限と優先度による足切り（D-010、2026-07-09）

- 本台帳（状態=ACTIVE）の**上限は20銘柄**。時間経過による自動失効は採用しない
  （WATCHに有効期限は設けない）。
- 投資判断部はWATCH判定時、判断メモに**上振れ余地スコア（高/中/低）**を明記する義務を負う
  （`playbooks/judgement.md` 参照）。本台帳の「優先度」列はそのスコアをそのまま転記する。
- ACTIVE銘柄が20銘柄を超えた場合、**週次ルーチン**（`corp/RUNBOOK.md`）で優先度が最も低い
  銘柄から順に`CLOSED`化し、日次巡回対象から除外する。同スコア内で複数銘柄が競合する場合は
  WATCH判定日が古いものを優先的に`CLOSED`化する。
- 優先度は固定ではない。投資判断部が既存WATCH銘柄を再評価した場合（トリガー成立時の更新等）、
  新しい判断メモのスコアで本台帳を更新する。
