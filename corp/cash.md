# 現金残高台帳（corp/cash.md）

初版作成: 2026-07-09（D-016）
位置づけ: D-015のポジションサイジング（比例ベット方式）が分母とする「資金残高＝現金＋保有
ポジション時価評価額」のうち、**現金部分**を記録する台帳。保有ポジションの時価は
`corp/portfolio.md` から計算できるが、現金はCEOの証券口座・銀行口座にしか実在しないため、
CEOからの申告のみが情報源となる（絶対制約第2条: 推測で埋めない）。

## 運用ルール

- **記録**: CEOが現金残高を報告した時点で、記録部（`playbooks/logging.md`）が1行追加する。
  報告は以下のタイミングで行う（CEOへの依頼事項）:
  1. 新規BUY判断メモの起案時（投資判断部がサイジング計算に使うため。報告が無い場合は
     直近の記録値を使い、記録が一件もない場合は当初資金300万円（D-002）をデフォルトとする
     — `playbooks/risk-check.md` 未確定事項の運用を正式化）。
  2. 入出金があった時（証券口座への追加入金・出金）。
  3. 少なくとも月1回、残高の棚卸し報告。
- **記録内容**: 報告日・現金残高・報告方法（チャットでの申告等）・備考（入出金があれば金額と
  理由）。数値はCEOの申告をそのまま転記し、エージェントが推測・補正しない。
- **行の削除はしない**（履歴として残す。最新行が現在値）。

## 資金残高（現金＋保有時価）の算出方法

`playbooks/risk-check.md`・`playbooks/judgement.md` のサイジング計算で使う分母:

```
資金残高 = 本台帳の最新現金残高 + corp/portfolio.md の状態=HELD全行の時価評価額合計
```

- 現金残高の報告が古い場合（例: 1ヶ月以上更新がない）、投資判断部は判断メモに「現金残高は
  YYYY-MM-DD時点の申告値であり、直近の入出金を反映していない可能性がある」旨を明記する。
- 資金残高が当初資金300万円から±20%を超えて乖離した場合、D-013の株価上限3,000円・単元
  チェックの前提を新しい決定IDで再計算する必要がある（D-015付随注記を参照）。
