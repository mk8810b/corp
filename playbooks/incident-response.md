# playbook: incident-response（インシデント対応）

推奨モデル階級: 重大度の判定・是正案の起案はSonnet以上（判断を伴うため）。検知自体は
どのバッジのエージェントが気づいても即座に発動してよい。

## 目的（D-016、2026-07-09新設）

絶対制約第2条は「出典を示せない数値・記憶ベースの数値の記載」「ハルシネーション」を
**重大事故**と定めるが、実際に発生した場合の検知後の対応手順がこれまで存在しなかった。
本playbookは、成果物（判断メモ・調査メモ・リスク照合結果・ダッシュボード等）に誤り
（数値の誤り・出典不備・ロジックの誤り・playbook違反等）が見つかった場合の対応を定める。

## 対象となる事象（重大度の分類）

- **CRITICAL**: 誤った数値・誤ったロジックが**CEOに既に提示された**（APPROVE済み判断メモ・
  発注検討材料として渡した情報）。CEOが誤った情報に基づいて意思決定するリスクがある。
- **MODERATE**: 誤りが社内工程内（校閲・機械照合等）で発見され、CEOに提示される前に
  止まった。
- **MINOR**: 誤字・体裁の誤り等、数値やロジックの正しさに影響しないもの。

## 対応手順

1. **即時停止・切り離し**: 誤りに気づいた時点で、その成果物に基づく後続処理（次工程への
   引き継ぎ・CEOへの新規提示）を中断する。**MINORを除き、まず訂正より先に停止を優先する**。
2. **CEOへの通知**（CRITICALは`PushNotification`で即時、MODERATEはその日のうちにチャットで
   報告、MINORは通常の成果物レビュー時に触れる程度でよい）。
3. **根本原因の特定**: どの工程（スクリーニング／企業調査／投資判断／リスク管理／記録／
   開発）で誤りが混入したか、なぜ校閲・機械照合をすり抜けたかを特定する。
4. **是正**: 誤った成果物は**上書きせず**、訂正版を新規に作成するか、既存ファイルに
   「訂正履歴」セクションを追記する形で対応する（過去の記録は変更・削除しないという
   憲法第2章-2の精神を、成果物レベルにも適用する）。何が・なぜ・いつ誤っていたかを
   訂正履歴に明記する。
5. **記録**: `corp/incidents.md` に1件追加する（下記参照）。
6. **再発防止（教訓化）**: 根本原因がplaybookの構造的な穴による場合、記録部の
   フィードバックループ（`playbooks/logging.md`・`corp/RUNBOOK.md`「記録部のフィードバック
   ループ」参照）と同じ手続きに従う: 是正案をCEOに提案し、**CEO承認を経て`corp/board.md`に
   新しい決定IDが付与されたものだけ**を該当playbookへ反映する。エージェントが独断で
   playbookを書き換えることは禁止する。

## チェックリスト

- [ ] 誤りに気づいた時点で後続処理を停止したか（MINORを除く）
- [ ] 重大度（CRITICAL/MODERATE/MINOR）を判定し、通知の緊急度を誤らせていないか
- [ ] 過去の成果物を無言で上書きせず、訂正履歴として残しているか
- [ ] `corp/incidents.md` に記録したか
- [ ] 再発防止策がplaybook変更を伴う場合、CEO承認・決定ID付与のプロセスを経ずに
      勝手に反映していないか
