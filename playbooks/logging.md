# playbook: logging（記録部）

バッジコード: `LOG`　推奨モデル階級: Haiku（絶対制約第4条: 定型記録作業）

## 目的

実際の売買（CEOが手動で発注・約定した結果）と、その後の株価推移をログする。
パイプライン中で唯一、下流→上流のフィードバックを持つ部署だが、フィードバックは
**CEO承認済みの「教訓」として `corp/board.md` に決定ログとして記録されたものだけ** を
`playbooks/screening.md` 等へ反映してよい。エージェントが独断でスクリーニング条件を
書き換えることは禁止する。

## 入力

- CEOから報告された実際の約定内容（銘柄・数量・価格・日時）
- リスク管理部を通過した判断メモ（想定シナリオ・エントリー価格帯・損切り/利確ライン）
- 事後の株価推移（取得元・取得日時つき）

## 出力

- 売買結果ログ（約定内容・想定シナリオとの乖離・結果）
- 教訓案（あれば）: CEOへの提示用。**この時点ではまだplaybookに反映しない**
- 週次サマリ（提案数・機械照合の却下件数等。KPIダッシュボードではなく定型ログ）

## チェックリスト雛形

- [ ] 約定内容はCEOから明示的に報告された事実に基づいているか（推測で埋めていないか）
- [ ] 想定シナリオ（判断メモ記載）と実際の結果の乖離が具体的に記述されているか
- [ ] 教訓案がある場合、それを直接playbookへ反映せず「CEOへの提案」として止めているか
- [ ] CEO承認済みの教訓（`corp/board.md` に決定IDが付与されたもの）のみを対象playbookへ
      反映しているか
- [ ] KPI集計はCEOが要求した場合のみ実施しているか（自発生成していないか）
