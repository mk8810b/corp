# playbook: judgement（投資判断部）

バッジコード: `JDG`　推奨モデル階級: 起案はSonnet、校閲はOpus（絶対制約第4条:
最終判断メモの校閲のみ上位モデル）

## 目的

企業調査部の調査メモを受けて、売買判断メモ（BUY/WATCH/PASS）を起案する。
エントリー価格帯・損切りライン・利確ラインを明記する。「1稿＋1校閲」で確定し、無限研磨はしない。

## BUY判定の基準（D-012、2026-07-09新設）

**「却下理由が一つも見つからなければBUY」という消極的な判定方式は禁止する。** 無料公開情報の
世界ではどの銘柄にも指摘可能なリスクが必ず一つはあり、逆に指摘可能なリスクが何もない銘柄は
既に適正価格が付いている（Fable5戦略レビュー §2-1）。BUY判定には、以下のうち**最低1つ以上**の
積極的な採用理由を、企業調査メモの事実に基づいて具体的に明記しなければならない:

- **カタリスト特定済み**: 業績・株価を動かす具体的な事象（決算・新製品・提携等）が特定でき、
  かつ市場にまだ十分織り込まれていないと判断できる根拠がある。
- **同業比バリュエーション下位**: 同業他社との比較で、PER/PBR等の指標が明確に割安であり、
  かつその割安さを正当化する構造的な欠陥（財務悪化・成長性喪失等）が見当たらない。
- **業績進捗が会社予想を上回る**: 直近の決算・開示が会社側の計画を上回るペースで進捗しており、
  かつ市場の反応がその進捗をまだ十分織り込んでいない。
- **エッジ仮説との整合**（D-012）: アナリストカバレッジが薄い銘柄の適時開示・決算短信を
  当日中に読み込み、公開情報の中でもまだ広く認知されていない具体的な事実（数値・記述）を
  発見できた。

上記に当てはまらない「単に欠点が見当たらない」という理由だけでのBUYは禁止する。
このリストは初期セットであり、実運転を通じて`corp/shadow-portfolio.md`の勝率データを見ながら
項目の追加・削除・重み付けを検討する（未確定事項参照）。

## トリガー文言の定量的な統制語彙（D-018、2026-07-09制定・同日検証済み）

WATCH判定の再評価トリガー（`corp/watchlist.md`）は、日次ルーチンで**自動化されたエージェントが
毎日判定する**ため、「明確に上抜け」等の曖昧な表現は日によって解釈がぶれる。以下の定量基準を
標準語彙として用いる（判断メモ起案時にこの語彙で書けば、日次ルーチンが機械的に判定できる）。
制定時の暫定値（±2%・1.5倍・5〜15%）はCEO指示による検証（ACTIVE WATCH 3銘柄の直近150営業日の
実データ。`outputs/review/2026-07-09-fable5-d018-trigger-validation.md`、D-018追記）を経て、
以下の通り確定した:

- **「（価格）を明確に上抜け/下抜け」**: **大引け後の確定終値**が、対象価格を
  **max(対象価格の2%, σ20, 呼び値1ティック)** 以上上回る/下回る。
  σ20＝当該銘柄の直近20営業日の日次終値変動率の標準偏差（出来高20日平均と同じ窓の時系列から
  Pythonで算出。憲法第2章-5）。
  - 根拠: 現行WATCH銘柄の実測で、2%は9432（日次σ0.85%）・9434（同0.97%）では約2σに相当し
    日常ノイズと判別可能（|日次変動|≥2%の日は各1.3%・3.4%、直近60日高値を2%超えた日は
    90判定日で0回）。σ20が2%を超える高ボラ銘柄では2%が1σ未満に落ちるためσ20を下限ガードとし、
    超低位株（8918: 呼び値1円≈株価の9〜10%、2%＝0.24円はサブティック）では1ティックに
    切り上げる（例: 8918-T3「年初来高値12円を上抜け」＝終値13円以上）。
  - 終値ベース判定の妥当性は実例で確認済み: 8918は2026-04-20にザラ場高値12円（年初来高値）を
    付けたが終値10円であり、終値ベースはこの日中スパイクを正しくノイズとして無視できた。
- **「出来高を伴って」**: 当日の**確定出来高**（大引け後）が、当該銘柄の直近20営業日平均出来高の
  **1.5倍以上**（検証の結果、維持）。
  - 必ず**自銘柄比の相対倍率**で判定する。東証全体の出来高ランキング・絶対株数での判定、
    および場中スナップショットでの判定は禁止（実例: 2026-07-09の8918は「出来高ランキング1位・
    約3.3億株」だったが、毎日約3億株が出来る銘柄であり自銘柄比1.18倍の平常日だった）。
  - 根拠: 実測で1.5倍以上は9432/9434とも130判定日中6日（4.6%）、8918は150日間で最大1.42倍と
    一度も未達であり、緩すぎる水準ではない（価格条件とのAND運用で誤検知率はさらに低い）。
    2.0倍への引き上げは9434で成立ゼロとなり、D-012（開示イベント当日読み込みがエッジ）の下では
    見逃しコストが誤検知コスト（イベント駆動ルーチン1回分のLLMコスト）を上回るため行わない。
  - 恒常的高出来高株（8918型）は倍率が構造的に1.5倍へ届きにくい（年初来高値を付けた日ですら
    1.42倍）。判断メモが根拠を明記すれば、銘柄別の明示倍率（例: 1.3倍以上）で上書きしてよい。
- **「押し目を形成し出来高が落ち着く」**: 判断メモに**具体値で明記された基準高値**からの下落率が
  終値ベースで**5〜15%の範囲に収まり**、かつ当日出来高が直近20営業日平均**以下**の日が
  **2営業日連続**した場合（成立日は2日目）。
  - 「直近高値」等の相対表現による基準高値の指定は禁止（バリュエーション数値明記ルールと同格）。
  - 注意: 本条件は「事件」ではなく「状態」であり、実測では60日高値基準で90判定日中13〜28日
    （2営業日連続条件でも6〜17日）成立する。急騰・高値形成が判断メモに記録された銘柄の
    沈静化確認という文脈でのみ用い、成立時は台帳ルール通り一度だけTRIGGERED化する。
  - 5〜15%という帯の幅自体は一般的な押し目慣行に由来する丸め値であり、統計的裏付けは
    示せていない（第2条に従い明記。シャドーポートフォリオのデータ蓄積後に再検証する）。
    なお帯上限15%は損切り-10%（D-002）より深いが、損切りはエントリー価格起点・本帯は
    基準高値起点であり矛盾しない（15%超は押し目ではなくトレンド転換とみなし成立させない）。
- **「バリュエーションが正常化」等の相対比較を伴う場合**: 判断メモが具体的な数値目標
  （例: PBR◯倍以下）を明記する（既存の慣行どおり。数値がない相対表現は禁止）。

**価格帯別の閾値分けは行わない**: D-009（下限100円）・D-013（上限3,000円）のスクリーニング
レンジでは呼び値は最大1円（JPX「呼値の単位」: 3,000円以下は「その他の銘柄」1円・TOPIX500構成
銘柄0.1〜0.5円。https://www.jpx.co.jp/equities/trading/domestic/07.html 、2025-05-02更新、
取得2026-07-09）であり、最悪ケース（100円・呼び値1円）でも1ティック＝1%＜2%。±2%は常に
1ティック以上で表現できるため単一基準でよい。レンジ外の既存WATCH銘柄（8918）は上記の
1ティック切り上げ則で処理する。

これらの基準値は3銘柄×150営業日の実測に基づく検証済み初期値である（検証の範囲と限界は
検証メモ§5参照。特にトリガー成立後の的中率は未検証）。シャドーポートフォリオ（D-012）の
データ蓄積後、新しい決定IDで再調整してよい。投資判断部は新規WATCH判定を書く際、独自の
曖昧な表現を作らずこの統制語彙（またはそれを上回る具体性を持つ数値条件）を使うこと。

## 入力

- `playbooks/research.md` の企業調査メモ
- `corp/board.md` D-002 のリスク上限値（1銘柄上限比率・損切り/利確ライン・セクター集中上限）
- `corp/board.md` D-011 のポートフォリオ構築方針（同時保有上限5銘柄・確信度連動サイジング・
  保有期間の扱い）
- `corp/portfolio.md`（現在の保有ポジション。BUY判定時に確信度・投入比率を検討する参考情報。
  最終的な保有銘柄数・セクター集中の機械照合はリスク管理部が行う）
- 直近の株価（取得元・取得日時つき）

## 出力

- 売買判断メモ（BUY / WATCH / PASS のいずれか）
  - エントリー価格帯
  - 損切りライン・利確ライン
  - 想定シナリオ（強気・弱気）
  - **BUY判定の場合は確信度（高 / 中 / 低のいずれか）と、対応する投入比率提案を必ず明記する**
    （D-011: 高=10%・中=5%・低=3%、D-002の1銘柄上限10%の範囲内）。確信度は企業調査メモの
    ファンダメンタルズの確度・上振れシナリオの規模感・下振れリスクの大きさを総合して判断する
    （出来高や値動きの強さだけで確信度を機械的に上げないこと）。
  - **投入比率の分母は現在資金残高**（現金＋保有ポジション時価評価額。当初資金300万円の
    固定額基準ではない — D-015比例ベット方式）。判断メモに投入額の円換算を書く場合は
    「現在資金残高×投入比率」でPython再計算し、分母に使った資金残高の値と算定日を併記する。
    単元による端数は原則切り捨て（1単元が目標額を超える場合の10%上限までの切り上げ許容は
    D-013の通り）。
  - **テール脆弱性フラグ（D-015）の該当有無を必ず確認・明記する**: (a)継続企業の前提に
    関する注記・重要事象 (b)直近の監査意見が無限定適正以外 (c)MSワラント（行使価額修正条項付
    新株予約権）等の大規模希薄化条項の未行使残存 (d)債務超過。**1つでも該当する銘柄は
    確信度「高」（10%枠）を提案できない**（上限は「中」=5%。損切り-10%が約定しない
    壊滅的ギャップ〈上場廃止等の-100%〉の資金全体への影響を5%以下に抑えるため）。
    該当有無は企業調査メモの記載事実に基づき判定し、判定根拠（記載箇所）を明記する。
  - **BUY判定の場合、保有期間は原則スイング（数週間〜数ヶ月）と明記する**（D-011）。長期保有への
    切替は投資判断部が独断で提案せず、CEOから個別の長期化指定があった場合のみ次回の判断メモ
    更新時に反映する。
  - **BUY判定の場合、シナリオ無効化条件を必ず明記する**（D-012）。「株価が損切りラインに
    達していなくても、買った理由（上記の採用理由）が崩れた場合は撤退する」という条件を、
    価格に依存しない形で具体的に書く（例: 「想定していたカタリストが不成立と判明した場合」
    「同業比較の前提となっていた財務指標が悪化した場合」等）。価格ラインより先にシナリオが
    崩れることの方が実務上多いため（Fable5戦略レビュー §7-3）、価格条件だけに頼らない。
  - **WATCH判定の場合は再評価トリガー条件を必ず明記する**（校閲確定・リスク管理部照合の後、
    週次ルーチン内で `corp/watchlist.md`（WATCH銘柄台帳）へ転記される。台帳の運用ルールは
    同ファイル冒頭を参照）
  - **WATCH判定の場合は上振れ余地スコア（高 / 中 / 低のいずれか）を必ず明記する**（D-010）。
    台帳が上限20銘柄を超えた際、週次ルーチンでこのスコアが最も低い銘柄から`CLOSED`化・
    巡回対象除外する優先度の基準として使う。判定根拠（企業調査メモの事実）と整合する形で
    スコアを付けること（機械的な数値だけでなく、上振れシナリオの確度・規模感を踏まえた
    定性判断でよい）。
  - front-matter必須項目:
    - `as_of`（分析基準日時）
    - `disclaimer`: 「本メモは情報提供・分析であり投資助言ではない。最終判断と結果責任はCEOに
      帰属する」（絶対制約第5条を必ずそのまま反映）
- 校閲後の最終版のみをリスク管理部へ渡す

- [ ] BUY判定の場合、保有期間がスイング前提で書かれているか（長期化はCEO個別指定のみ、D-011）
- [ ] BUY判定の場合、積極的な採用理由が最低1つ以上、企業調査メモの事実に基づいて具体的に
