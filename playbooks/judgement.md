# playbook: judgement（投資判断部）

バッジコード: `JDG`　推奨モデル階級: 起案はSonnet、校閲はOpus（絶対制約第4条:
最終判断メモの校閲のみ上位モデル）

## 目的

企業調査部の調査メモを受けて、売買判断メモ（BUY/WATCH/PASS）を起案する。
エントリー価格帯・損切りライン・利確ラインを明記する。「1稿＋1校閲」で確定し、無限研磨はしない。

## BUY判定の基準（D-012、2026-07-09新設）

**「却下理由が一つも見つからなければBUY」という消極的な判定方式は禁止する。** 無料公開情報の
世界ではどの銘柄にも指摘可能なリスクが必ず一つはあり、逆に指摘可能なリスクが何もない銘柄は
既に適正価格が付いている（Fable5戦略レビュー §2-1）。BUY判定には、以下のうち**最低1つ以上**の
積極的な採用理由を、企業調査メモの事実に基づいて具体的に明記しなければならない:

- **カタリスト特定済み**: 業績・株価を動かす具体的な事象（決算・新製品・提携等）が特定でき、
  かつ市場にまだ十分織り込まれていないと判断できる根拠がある。
- **同業比バリュエーション下位**: 同業他社との比較で、PER/PBR等の指標が明確に割安であり、
  かつその割安さを正当化する構造的な欠陥（財務悪化・成長性喪失等）が見当たらない。
- **業績進捗が会社予想を上回る**: 直近の決算・開示が会社側の計画を上回るペースで進捗しており、
  かつ市場の反応がその進捗をまだ十分織り込んでいない。
- **エッジ仮説との整合**（D-012）: アナリストカバレッジが薄い銘柄の適時開示・決算短信を
  当日中に読み込み、公開情報の中でもまだ広く認知されていない具体的な事実（数値・記述）を
  発見できた。

上記に当てはまらない「単に欠点が見当たらない」という理由だけでのBUYは禁止する。
このリストは初期セットであり、実運転を通じて`corp/shadow-portfolio.md`の勝率データを見ながら
項目の追加・削除・重み付けを検討する（未確定事項参照）。

## 入力

- `playbooks/research.md` の企業調査メモ
- `corp/board.md` D-002 のリスク上限値（1銘柄上限比率・損切り/利確ライン・セクター集中上限）
- `corp/board.md` D-011 のポートフォリオ構築方針（同時保有上限5銘柄・確信度連動サイジング・
  保有期間の扱い）
- `corp/portfolio.md`（現在の保有ポジション。BUY判定時に確信度・投入比率を検討する参考情報。
  最終的な保有銘柄数・セクター集中の機械照合はリスク管理部が行う）
- 直近の株価（取得元・取得日時つき）

## 出力

- 売買判断メモ（BUY / WATCH / PASS のいずれか）
  - エントリー価格帯
  - 損切りライン・利確ライン
  - 想定シナリオ（強気・弱気）
  - **BUY判定の場合は確信度（高 / 中 / 低のいずれか）と、対応する投入比率提案を必ず明記する**
    （D-011: 高=10%・中=5%・低=3%、D-002の1銘柄上限10%の範囲内）。確信度は企業調査メモの
    ファンダメンタルズの確度・上振れシナリオの規模感・下振れリスクの大きさを総合して判断する
    （出来高や値動きの強さだけで確信度を機械的に上げないこと）。
  - **投入比率の分母は現在資金残高**（現金＋保有ポジション時価評価額。当初資金300万円の
    固定額基準ではない — D-015比例ベット方式）。判断メモに投入額の円換算を書く場合は
    「現在資金残高×投入比率」でPython再計算し、分母に使った資金残高の値と算定日を併記する。
    単元による端数は原則切り捨て（1単元が目標額を超える場合の10%上限までの切り上げ許容は
    D-013の通り）。
  - **テール脆弱性フラグ（D-015）の該当有無を必ず確認・明記する**: (a)継続企業の前提に
    関する注記・重要事象 (b)直近の監査意見が無限定適正以外 (c)MSワラント（行使価額修正条項付
    新株予約権）等の大規模希薄化条項の未行使残存 (d)債務超過。**1つでも該当する銘柄は
    確信度「高」（10%枠）を提案できない**（上限は「中」=5%。損切り-10%が約定しない
    壊滅的ギャップ〈上場廃止等の-100%〉の資金全体への影響を5%以下に抑えるため）。
    該当有無は企業調査メモの記載事実に基づき判定し、判定根拠（記載箇所）を明記する。
  - **BUY判定の場合、保有期間は原則スイング（数週間〜数ヶ月）と明記する**（D-011）。長期保有への
    切替は投資判断部が独断で提案せず、CEOから個別の長期化指定があった場合のみ次回の判断メモ
    更新時に反映する。
  - **BUY判定の場合、シナリオ無効化条件を必ず明記する**（D-012）。「株価が損切りラインに
    達していなくても、買った理由（上記の採用理由）が崩れた場合は撤退する」という条件を、
    価格に依存しない形で具体的に書く（例: 「想定していたカタリストが不成立と判明した場合」
    「同業比較の前提となっていた財務指標が悪化した場合」等）。価格ラインより先にシナリオが
    崩れることの方が実務上多いため（Fable5戦略レビュー §7-3）、価格条件だけに頼らない。
  - **WATCH判定の場合は再評価トリガー条件を必ず明記する**（校閲確定・リスク管理部照合の後、
    週次ルーチン内で `corp/watchlist.md`（WATCH銘柄台帳）へ転記される。台帳の運用ルールは
    同ファイル冒頭を参照）
  - **WATCH判定の場合は上振れ余地スコア（高 / 中 / 低のいずれか）を必ず明記する**（D-010）。
    台帳が上限20銘柄を超えた際、週次ルーチンでこのスコアが最も低い銘柄から`CLOSED`化・
    巡回対象除外する優先度の基準として使う。判定根拠（企業調査メモの事実）と整合する形で
    スコアを付けること（機械的な数値だけでなく、上振れシナリオの確度・規模感を踏まえた
    定性判断でよい）。
  - front-matter必須項目:
    - `as_of`（分析基準日時）
    - `disclaimer`: 「本メモは情報提供・分析であり投資助言ではない。最終判断と結果責任はCEOに
      帰属する」（絶対制約第5条を必ずそのまま反映）
- 校閲後の最終版のみをリスク管理部へ渡す

## チェックリスト雛形（起案）

- [ ] BUY/WATCH/PASSのいずれかが明記されているか
- [ ] WATCH判定の場合、再評価トリガー条件（イベント・価格条件等）が明記されているか
      （`corp/watchlist.md` への転記に耐える具体性があるか）
- [ ] WATCH判定の場合、上振れ余地スコア（高/中/低）が明記され、判断根拠と整合しているか（D-010）
- [ ] BUY判定の場合、確信度（高/中/低）と対応する投入比率（10%/5%/3%）が明記されているか（D-011）
- [ ] BUY判定の場合、テール脆弱性フラグ4項目（D-015）の該当有無と判定根拠が明記されているか。
      該当があるのに確信度「高」を提案していないか
- [ ] 投入額の円換算がある場合、分母が現在資金残高（D-015）でPython再計算されているか
- [ ] BUY判定の場合、保有期間がスイング前提で書かれているか（長期化はCEO個別指定のみ、D-011）
- [ ] BUY判定の場合、積極的な採用理由が最低1つ以上、企業調査メモの事実に基づいて具体的に
      明記されているか（「却下理由がないから」という消極的判定になっていないか、D-012）
- [ ] BUY判定の場合、価格に依存しないシナリオ無効化条件が明記されているか（D-012）
- [ ] エントリー価格帯・損切りライン・利確ラインが具体的な数値で書かれているか
- [ ] 使用した数値に取得元・取得日時が付いているか（企業調査メモからの引き写しも含めて再確認）
- [ ] front-matterに `as_of` と第5条の免責文言があるか
- [ ] 発注・約定に関する記述（証券会社API呼び出し等）が一切含まれていないか（第1条）

## 未確定事項（実運転しながらD-logで固める）

- WATCH銘柄の再評価トリガーの定量細則（「明確に上抜け」等の判定基準）
  （台帳への記録経路・容量上限・優先度基準はD-010・`corp/watchlist.md` 新設により整備済み・2026-07-09）

  仮想勝率データが蓄積してから検証する。項目ごとの的中率が計測できるようになれば、
  弱い項目の廃止・強い項目への重み付けを検討する。
