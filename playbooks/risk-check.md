# playbook: risk-check（リスク管理部）

バッジコード: `RSK`　推奨モデル階級: Haiku可（絶対制約第4条: ルールベースの機械照合のため）

## 目的

投資判断部の全ての売買判断メモを、`corp/board.md`（D-002）のリスク上限値と機械的に照合する。
自己申告ではなく、数値の再計算・照合は必ずスクリプト（Python）で行う。
逸脱する提案は、理由の如何を問わず**提案自体を却下**する（絶対制約第3条）。

## 入力

- `playbooks/judgement.md` の校閲済み最終判断メモ
- `corp/board.md` D-002 のリスク上限値（現在値は下記に転記。正本はboard.md）
  - 1銘柄あたり最大投入比率: 10%
  - 損切りライン: -10%
  - 利確ライン: +25%
  - 同一セクター集中上限: 40%
- `corp/board.md` D-011 のポートフォリオ構築方針（現在値は下記に転記。正本はboard.md）
  - 同時保有銘柄数上限: 5銘柄
  - ポジションサイジング: 確信度連動（高10%・中5%・低3%）が投入比率提案と整合しているか
- `corp/board.md` D-015 のポートフォリオレベル上限（現在値は下記に転記。正本はboard.md）
  - 投入比率の分母: **現在資金残高**（現金＋保有ポジション時価評価額。当初資金300万円の
    固定額基準ではない。比例ベット方式）
  - ポートフォリオ想定最大同時損失（PML）上限: **資金の5.0%**
    （PML = 状態HELD全ポジション＋新規提案の Σ(投入比率×損切り幅10%)）
  - テール脆弱性フラグ該当銘柄の投入比率上限: **5%（確信度「中」相当）**。フラグは
    (a)継続企業の前提に関する注記・重要事象 (b)監査意見が無限定適正以外
    (c)MSワラント等の大規模希薄化条項の未行使残存 (d)債務超過 の4項目（企業調査メモの
    記載から判定。裁量による追加・免除はしない）
- `corp/board.md` D-013（単元株フィージビリティ）: 株価上限3,000円（スクリーニング段階の
  フィルタ）。ただし既存WATCH銘柄からのBUY起案等、フィルタをすり抜けた提案に備え、
  下記の単元購入額チェックを機械照合でも行う。
- `corp/portfolio.md`（現在の保有ポジション一覧。セクター集中判定・保有銘柄数判定に使用）

## 統制語彙（2026-07-09改定）

機械照合結果は **APPROVE / REJECT** の2値のみを使用する。

- **APPROVE** = 照合通過（リスク上限に抵触せず、CEOへ提示可）
- **REJECT** = リスク上限に抵触（提案自体を却下。CEOへ提示しない）

リスク管理部では「PASS」という語を**使用禁止**とする。投資判断部の判定語「PASS」
（BUY/WATCH/PASSの一つ、＝見送り）と正反対の意味で衝突し、CEOの読み違え事故の芽になるため
（2026-07-09の独立レビュー §1-1 の指摘によりplaybook改定）。

## 出力

- 機械照合結果（APPROVE / REJECT）と、REJECTの場合は逸脱項目・数値の明記
- APPROVEされた判断メモのみCEOへ提示する

## チェックリスト雛形

- [ ] 1銘柄あたりの投入比率上限（10%）を超えていないか（Pythonで再計算）
- [ ] 投入比率の金額換算の分母が**現在資金残高**（現金＋保有ポジション時価評価額、
      判断メモ起案時点の台帳記録値）になっているか（D-015。当初資金300万円固定を分母に
      していないか。Pythonで再計算）
- [ ] BUY判定の場合、投入比率が確信度（高/中/低）に対応する値（10%/5%/3%）と一致しているか
      （D-011。確信度より高い比率を提案していないか）
- [ ] BUY判定の場合、テール脆弱性フラグ（D-015: 継続前提注記／監査意見が無限定適正以外／
      MSワラント等の希薄化条項残存／債務超過）への該当有無を企業調査メモと照合したか。
      **フラグ該当銘柄への確信度「高」（10%枠）提案はREJECT**（5%以下なら該当してもよい）
- [ ] 新規BUY提案を加えた**PML（Σ 投入比率×10%）が資金の5.0%以下**か（D-015。
      `corp/portfolio.md` の状態=HELD全行込みでPythonで再計算）
- [ ] BUY判定の場合、単元（100株）購入額を計算し、確信度連動の投入比率目標額を超えていないか
      確認する（D-013）。超えている場合、D-002の1銘柄上限10%（運用資金の10%）までの
      切り上げは許容するが、単元購入額が10%上限すら超える場合は**REJECT**とする（買えない
      提案をCEOに提示しない）。Pythonで再計算すること。
- [ ] 損切りライン（-10%）・利確ライン（+25%）がメモに明記されており、上限と矛盾していないか
- [ ] 提案銘柄を加えた場合の同一セクター集中比率が上限（40%）を超えないか（`corp/portfolio.md`
      の保有ポジション込みでPythonで再計算）
- [ ] BUY判定の場合、`corp/portfolio.md` の保有銘柄数（状態=HELD）に本提案を加えて
      **5銘柄を超えないか**（D-011。超える場合はREJECT）
- [ ] リスク上限値は `corp/board.md` の最新の決定ログ（最大のD番号）から転記されているか
      （古い数値を使っていないか）
- [ ] 照合結果の表記は統制語彙（APPROVE / REJECT）のみを使用しているか
      （「PASS」を機械照合結果として使用していないか）
- [ ] REJECTの場合、逸脱項目・数値・上限値を具体的に記録しているか
- [ ] APPROVEされた判断メモのみをCEO提示対象としているか（REJECT案が混入していないか）

## 未確定事項（実運転しながらD-logで固める）

- セクター分類の基準（業種コード体系）
- 複数銘柄が同時にセクター上限に抵触する場合の優先順位付けルール
- 現在資金残高（D-015の分母）のうち**現金部分の記録場所**が未整備。保有ポジションの
  時価は `corp/portfolio.md` から再計算できるが、現金残高はCEOの口座にしか存在しない。
  当面は判断メモ起案時にCEOから現金残高の報告を受けて分母を確定する（報告値と報告日時を
  メモに記録 — 第2条）。台帳への常設（portfolio.mdへの現金行追加等）はD-logで固める。
  保有0件かつ報告がない場合は当初資金300万円（D-002）を分母のデフォルトとしてよい。
