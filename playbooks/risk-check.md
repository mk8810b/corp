# playbook: risk-check（リスク管理部）

バッジコード: `RSK`　推奨モデル階級: Haiku可（絶対制約第4条: ルールベースの機械照合のため）

## 目的

投資判断部の全ての売買判断メモを、`corp/board.md`（D-002）のリスク上限値と機械的に照合する。
自己申告ではなく、数値の再計算・照合は必ずスクリプト（Python）で行う。
逸脱する提案は、理由の如何を問わず**提案自体を却下**する（絶対制約第3条）。

## 入力

- `playbooks/judgement.md` の校閲済み最終判断メモ
- `corp/board.md` D-002 のリスク上限値（現在値は下記に転記。正本はboard.md）
  - 1銘柄あたり最大投入比率: 10%
  - 損切りライン: -10%
  - 利確ライン: +25%
  - 同一セクター集中上限: 40%
- `corp/board.md` D-011 のポートフォリオ構築方針（現在値は下記に転記。正本はboard.md）
  - 同時保有銘柄数上限: 5銘柄
  - ポジションサイジング: 確信度連動（高10%・中5%・低3%）が投入比率提案と整合しているか
- `corp/portfolio.md`（現在の保有ポジション一覧。セクター集中判定・保有銘柄数判定に使用）

## 統制語彙（2026-07-09改定）

機械照合結果は **APPROVE / REJECT** の2値のみを使用する。

- **APPROVE** = 照合通過（リスク上限に抵触せず、CEOへ提示可）
- **REJECT** = リスク上限に抵触（提案自体を却下。CEOへ提示しない）

リスク管理部では「PASS」という語を**使用禁止**とする。投資判断部の判定語「PASS」
（BUY/WATCH/PASSの一つ、＝見送り）と正反対の意味で衝突し、CEOの読み違え事故の芽になるため
（2026-07-09の独立レビュー §1-1 の指摘によりplaybook改定）。

## 出力

- 機械照合結果（APPROVE / REJECT）と、REJECTの場合は逸脱項目・数値の明記
- APPROVEされた判断メモのみCEOへ提示する

## チェックリスト雛形

- [ ] 1銘柄あたりの投入比率上限（10%）を超えていないか（Pythonで再計算）
- [ ] BUY判定の場合、投入比率が確信度（高/中/低）に対応する値（10%/5%/3%）と一致しているか
      （D-011。確信度より高い比率を提案していないか）
- [ ] 損切りライン（-10%）・利確ライン（+25%）がメモに明記されており、上限と矛盾していないか
- [ ] 提案銘柄を加えた場合の同一セクター集中比率が上限（40%）を超えないか（`corp/portfolio.md`
      の保有ポジション込みでPythonで再計算）
- [ ] BUY判定の場合、`corp/portfolio.md` の保有銘柄数（状態=HELD）に本提案を加えて
      **5銘柄を超えないか**（D-011。超える場合はREJECT）
- [ ] リスク上限値は `corp/board.md` の最新の決定ログ（最大のD番号）から転記されているか
      （古い数値を使っていないか）
- [ ] 照合結果の表記は統制語彙（APPROVE / REJECT）のみを使用しているか
      （「PASS」を機械照合結果として使用していないか）
- [ ] REJECTの場合、逸脱項目・数値・上限値を具体的に記録しているか
- [ ] APPROVEされた判断メモのみをCEO提示対象としているか（REJECT案が混入していないか）
